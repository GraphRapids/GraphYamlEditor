# ADR-001: Standardised Dockerfile, Health-Check, and Integration Test Conventions

## Status

Accepted

## Context

GraphRapids is introducing pre-push integration validation via Graphras.
Each service/component repository needs a Docker image, a health-check
endpoint, and integration tests that can be executed against a live
instance.  Without a shared convention, each repository would make ad-hoc
choices that complicate orchestration.

## Decision

### Dockerfile

- Every repository MUST provide a `Dockerfile` at the repo root.
- The Dockerfile MUST use a **multi-stage build**: the final stage must not
  contain dev dependencies, source code, or build tooling.
- The final stage SHOULD be based on a minimal base image (e.g.
  `node:20-alpine`, `python:3.12-slim`).
- An `EXPOSE <port>` directive MUST be present and match the port used in
  `docker-compose.yml`.
- A `HEALTHCHECK` instruction MUST be included.

### Health-Check Endpoint

- Every service MUST expose `GET /health`.
- The response MUST be HTTP 200 with body `{"status":"ok"}` and
  `Content-Type: application/json`.
- No authentication is required for this endpoint.
- If the service has hard dependencies (e.g. a database), the health check
  SHOULD verify connectivity; otherwise a simple static response is
  sufficient.

### Docker Compose

- A `docker-compose.yml` MUST be present at the repo root.
- It MUST include a `healthcheck` block so that `docker compose up --wait`
  reliably gates on readiness.
- Port mappings MUST match the `EXPOSE` directive.

### Integration Tests

- Integration tests MUST live in `tests/integration/`.
- They MUST be runnable via a dedicated script (e.g. `npm run
  test:integration`) that does NOT overlap with the unit test command.
- Tests operate **purely over HTTP** against a configurable `SERVICE_URL`
  environment variable (default `http://localhost:<port>`).
- Tests MUST NOT import application internals or share process memory with
  the service under test.
- Tests MUST be self-contained: they create their own data and clean up
  afterwards.

### Secrets

- No secrets, credentials, or API keys may be hard-coded in the Dockerfile,
  docker-compose.yml, or integration tests.
- All sensitive configuration MUST be injected via environment variables at
  runtime.

### Port Allocation

- Each service uses a unique port to allow local composition of the full
  suite.  Allocated ports are tracked in this ADR.

| Service            | Port |
| ------------------ | ---- |
| GraphYamlEditor    | 6080 |

## Consequences

- Graphras can bring up any service with `docker compose up --wait` and
  know exactly how to poll readiness and run integration tests.
- Developers can run integration tests locally with the same commands CI
  uses.
- The multi-stage build convention keeps production images small and free of
  dev tooling.
- New repositories must follow this ADR when adding Docker support.
